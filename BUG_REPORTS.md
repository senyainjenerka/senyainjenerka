## Search & Browse

### BUG-001. Search query parameters are missing in URL
**Окружение:** Stage, Chrome 149
**Предусловия:** пользователь разлогинен
**Шаги воспроизведения:**
1. [открыта нужная страница приложения]
2. Ввести корректные критерии поиска и нажать иконку «Search»
3. Скопировать URL открывшейся страницы
4. Вставить URL в новую вкладку браузера
5. Сравнить результаты поиска на старой и новой странице

**Фактический результат:** на новой странице отображается плашка «No lawyers found»
**Ожидаемый результат:** результаты поиска должны быть идентичны при переходе по скопированному URL
**Требования:** [см. документацию требований проекта] (обсуждение с бизнес-аналитиком и PO — в комментариях к тикету)

### BUG-003. "Back" button redirects to sign-up form instead of search results list
**Окружение:** Stage, Google Chrome 148.0.7778.218
**Предусловия:** [открыта нужная страница приложения], пользователь не залогинен
**Шаги воспроизведения:**
1. Ввести practice area/имя и штат
2. Нажать кнопку «Search»
3. Выбрать специалиста и открыть его полный профиль
4. Нажать кнопку «Назад» в браузере

**Фактический результат:** кнопка «Назад» перенаправляет на форму регистрации вместо списка результатов поиска
**Ожидаемый результат:** кнопка «Назад» должна возвращать к списку результатов поиска с сохранёнными ранее применёнными критериями поиска и фильтрами

### BUG-005. Filters panel disappears when scrolling through the Pro list instead of staying fixed
**Окружение:** Stage, Google Chrome 148.0.7778.218
**Предусловия:**
1. [открыта нужная страница приложения]
2. В результатах отображается хотя бы один специалист
3. Отображена панель фильтров

**Шаги воспроизведения:**
1. Прокрутить список специалистов вниз

**Фактический результат:** панель фильтров видна только в верхней части страницы; при прокрутке вниз она исчезает, остаётся только список специалистов
**Ожидаемый результат:** панель фильтров должна быть зафиксирована на экране при прокрутке списка, чтобы пользователь мог менять фильтры в любой момент без прокрутки наверх
**Требования:** [см. документацию требований проекта]

## Profile

### BUG-006. Text in "Add credit card" and "Add bank account" buttons overflows the button border at viewport less than 938 px
**Окружение:** Stage, Google Chrome 148.0.7778.218, DevTools
**Предусловия:** [открыта нужная страница приложения], пользователь залогинен
**Шаги воспроизведения:**
1. Открыть иконку профиля
2. Выбрать раздел «payments»
3. В DevTools (режим адаптивности) установить ширину вьюпорта 938 px или меньше

**Фактический результат:** текст внутри кнопок «Add credit card» и «Add bank account» выходит за границы кнопки
**Ожидаемый результат:** текст кнопок должен оставаться внутри границ кнопки при любом размере экрана

### BUG-007. Sidebar menu stays open and covers half the screen on mobile
**Окружение:** Stage, Google Chrome 148.0.7778.218, DevTools
**Предусловия:** [открыта нужная страница приложения], пользователь залогинен
**Шаги воспроизведения:**
1. Открыть иконку профиля
2. Выбрать раздел «profile»
3. В DevTools установить мобильную ширину вьюпорта, например 491 px

**Фактический результат:** боковое меню остаётся открытым и не сворачивается; чтобы увидеть и взаимодействовать с контентом страницы, пользователю приходится скроллить по горизонтали
**Ожидаемый результат:** на мобильном экране боковое меню должно сворачиваться в гамбургер-меню либо быть скрыто по умолчанию, чтобы основной контент занимал всю ширину экрана и был полностью читаем без горизонтальной прокрутки

### BUG-008. Card details overflow the card container block at viewport less than 689 px
**Окружение:** Stage, Google Chrome 148.0.7778.218, DevTools
**Предусловия:** [открыта нужная страница приложения], пользователь залогинен
**Шаги воспроизведения:**
1. Открыть иконку профиля
2. Выбрать раздел «payments»

**Ожидаемый результат:** данные карты должны полностью помещаться в контейнер при любом размере экрана
